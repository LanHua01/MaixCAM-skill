# Serial Protocol Decision Template

用户选择方案后再填写。任何候选字段、心跳或 ACK 都不是本 skill 的默认协议。

## 物理链路

- 通信对象与数据方向：
- UART1（推荐）或 UART0（已确认启动风险）：
- 端口、TX/RX 引脚、逻辑电平、共地：
- 波特率、数据位、停止位、校验位：
- 是否存在启动日志、Type-C 方向或电平转换风险：

## 数据语义

- 需要发送/接收的数据：
- 坐标原点、单位、正方向、数值精度、时间戳：
- 目标状态：`visible/cached/lost`、置信度、错误码：
- 下位机如何处理过期、低置信度和丢失目标：

## 候选帧比较

| 选项 | 适用条件 | 代价 | 是否被选中 |
| --- | --- | --- | --- |
| 文本帧 | 调试优先、人眼可读 | 带宽较大 |  |
| 固定/变长二进制 | 高频或严格字段 | 需明确字节序与校验 |  |
| JSON/CSV | 字段变动较多 | 长度与解析开销 |  |
| 题目/下位机已有协议 | 已有明确约束 | 兼容与测试成本 |  |

## 最终分包与调度

- 帧头/帧尾/长度/字节序/校验/转义/最大帧长：
- 粘包、拆包、超时、解析失败和缓冲溢出策略：
- 固定周期、状态变化或查询响应：
- ACK、心跳、序号、重发是否需要及其题面依据：
- 最终协议示例和串口助手验证记录：
