# Task Intake Questions

先读取用户上传的题面、图片、接线图、协议和模型；只追问缺失项。不要一次机械问完所有问题，优先问会改变方案的高影响事实。第一阶段未完成时，不生成最终工程、固定协议或 MaixPy 代码。

## 第一阶段：首次配置门禁

### 题目与验收

1. 题型、完整任务目标和评分指标是什么？是否有题面或样例？
2. 识别对象、输出对象、精度、时限、误检/漏检代价和失效动作是什么？
3. 是否限制联网、PC、相机数量、补光、模型、无线通信或外部传感器？
4. 是否已有目标 FPS、最大端到端时延、连续运行时长或允许的性能降级？

### MaixCAM、相机与屏幕

1. MaixCAM Pro 的系统/固件版本、相机型号、镜头、分辨率、安装高度和方向是什么？
2. 是否需要屏幕 UI？屏幕型号、宽高像素、显示方向、是否需要触摸调参？
3. 若需要 UI：必须显示哪些画面、数值、目标状态、通信状态、错误、性能信息和调参入口？刷新频率或实时性有什么要求？
4. 是否需要现场运行/调参 UI 配置档？触摸屏、按键或串口命令中哪些输入已实际确认可用？若没有输入设备，采用无 UI 或何种替代观测方式？
4. 场地光照、背景、目标颜色/尺寸、距离和安装方式是否固定？

### 接口与安全

1. 下位机、执行机构、通信接口、具体引脚、逻辑电平和供电方式是什么？
2. 是否已有接线图、端口、波特率、协议或执行机构保护逻辑？
3. 是否需要心跳、ACK、序号、重发、查询响应或超时检测？已有时请提供周期、超时动作和下位机处理方式。
4. 是否需要急停、限幅、低速模式、激光/飞行安全联锁？

### MaixVision 调试补充（仅在涉及 IDE、部署或运行异常时）

1. MaixVision、固件/MaixPy、板卡型号和主机系统版本分别是什么？
2. 通过何种方式连接设备；错误出现于连接、在线运行、完整项目运行、预览、上传、打包还是安装？
3. 请提供完整控制台日志、最小复现步骤，以及单文件还是包含模型/资源的项目。
4. 资源是否已部署到设备侧；电脑端的红线/无补全是否与板端实际运行错误同时出现？
5. 是否要求交付可直接打开的完整工程文件夹？工程名称、入口、模型/资源文件和在线运行或打包方式是否已有约定？

## 第二阶段：方案确认

第一阶段完成后，先输出 `solution-options.md` 的方案矩阵，等待用户选择后再问：

1. 最终采用哪套识别路线及其降级路线？
2. 最终协议、字段、单位、频率、ACK/心跳和异常策略是什么？
3. 是否最终需要屏幕 UI？若需要，必须显示哪些画面、数值、状态、错误、性能和调参入口？
4. 心跳、ACK、序号、重发、超时和丢失策略是否需要；其周期、触发条件和下位机动作是什么？
5. 各模块的输入/输出、独立测试脚本和验收条件是什么？
6. 若使用 YOLO，是否已有图片、标注或 `.mud` 模型？没有时是否先执行数据采集与训练计划？
