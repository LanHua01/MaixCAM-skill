# Task Intake Questions

使用 skill 时，先用这些问题收敛题目约束。不要一次机械全问；优先问会改变方案的高影响问题。

## 题目目标

1. 这道题最终要完成什么动作或测量？
2. 视觉需要识别哪些对象？
3. 视觉输出给谁使用？
4. 评分更看重精度、速度、稳定性、成功次数，还是抗干扰？

## 场地和目标

1. 目标颜色、形状、尺寸是否固定？
2. 背景颜色和材质是什么？
3. 相机到目标距离是否固定？
4. 目标是否运动？速度大概多少？
5. 是否允许补光、遮光、固定曝光？

## 硬件

1. MaixCAM Pro 使用哪个摄像头/镜头？
2. 相机固定还是跟随机构运动？
3. 下位机型号是什么？
4. 执行机构是什么：舵机、步进、电机、机械臂、绳驱、小车？
5. 是否已有接线和串口端口约定？

## 算法路线

1. 目标是否可以用颜色阈值稳定识别？
2. 是否存在黑线、矩形、圆、棋盘等几何约束？
3. 是否需要识别类别语义，必须使用 YOLO 或分类模型？
4. 是否已有 `.mud` 模型？
5. 误检和漏检哪个代价更高？

## 坐标和控制

1. 输出像素坐标还是物理坐标？
2. 原点在哪里？
3. x/y 正方向如何定义？
4. 是否需要比例尺或标定点？
5. 下位机需要坐标、误差、速度、还是事件命令？

## 通信和协议

1. 通信方式是什么：UART、I2C、SPI、网络、文件？
2. 是否已有协议格式？
3. 是否需要帧头/帧尾/长度/校验？
4. 是否需要心跳、ACK、序号或重发？
5. 发送周期由 MaixCAM 固定发送，还是下位机查询？

## 验收

1. 单独视觉成功的判据是什么？
2. 通信成功的判据是什么？
3. 闭环成功的判据是什么？
4. 现场调试时最需要显示哪些状态？
5. 失败时应该停止、保持、重试还是切换搜索？
