# Acceptance Checklist

按实际题目裁剪。验收顺序为：独立硬件与视觉 → 通信 → 低速闭环 → 全场景与公开发布检查。

## 相机、显示与视觉

- [ ] 焦距、曝光、白平衡、分辨率和帧率已在赛场环境验证。
- [ ] 屏幕像素、方向、触摸映射和关键状态显示正确。
- [ ] 候选、确认、缓存、丢失状态及失效动作明确；确认/丢失阈值与目标选择规则已验证。
- [ ] ROI、阈值、尺寸、几何或标定误差已用题目指标验证。
- [ ] 单帧异常不会直接触发危险动作。

## 模型

- [ ] `.mud` 文件、类别顺序、输入尺寸和图像格式一致。
- [ ] 数据来源和标注许可已确认；训练/验证/赛场测试没有混淆。
- [ ] 置信度、IOU、ROI、尺寸与时序过滤已在现场测试。
- [ ] `dual_buff=True` 的一帧结果延迟已被允许、补偿，或已关闭。
- [ ] 模型加载失败、低置信度或目标丢失时有安全降级。

## 通信与闭环

- [ ] GND 共地、RX/TX 交叉、逻辑电平、端口和波特率正确。
- [ ] UART0 使用时已处理启动日志和 A16 上电风险；否则使用 UART1。
- [ ] 解析器支持粘包、拆包、超时、异常字节和日志限流。
- [ ] 协议、字段、单位、频率、ACK/心跳由用户或题面确认。
- [ ] 已有工程的已确认引脚、协议、波特率和安全联锁未被无故替换；固定测试包与下位机解析已先行验证。
- [ ] 单轴/单机构低速、方向、限幅、急停和目标 lost 策略通过。

## 工程化与公开发布

- [ ] 主循环、任务、通信、显示、配置和独立测试职责清晰，无无意义重复逻辑。
- [ ] 相机、算法、串口、协议和执行机构均可单独验证。
- [ ] 生成的工程代码注释为中文，且只保留必要的职责、硬件、单位、安全与时序说明；未改写 Python 或 MaixPy API 名称。
- [ ] 正向代码示例已对照 MaixPy 官方文档；未使用 OpenMV/K210/PC 专属 API。
- [ ] 输出前已检查 MaixVision/板端运行时边界、Python 语法、UART 字段数和版本敏感 API 假设。
- [ ] 涉及 MaixVision 时，已分别验证设备连接、最小相机显示、完整项目资源、在线运行与打包/安装后的行为。
- [ ] 已交付唯一 `main.py` 入口和完整工程目录；在 MaixVision 打开文件夹并运行完整项目后可进入可观察的调试状态。
- [ ] 多文件、模型和资源均随工程提供并使用相对路径；缺失模型/资源时显示安全降级，而非伪造最终可运行结果。
- [ ] 已在目标 MaixCAM/MaixCAM Pro 记录相机、视觉、模型、UI/串口、闭环和连续运行的性能；未实测项明确标为待实测。
- [ ] 性能问题已附复现条件、日志、瓶颈判断、优化代价和精度/时序/安全回归结果。
- [ ] 发布文本不含姓名、账号、邮箱、手机号、本机路径、令牌、密码、私钥、内网地址或未经授权的数据集。
- [ ] 外部链接可公开访问，模型、图片和标注的来源及许可已确认。
