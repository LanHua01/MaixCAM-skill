# Acceptance Checklist

用于项目交付前检查。按题目删减即可。

## 相机和显示

- [ ] 画面清晰，对焦稳定。
- [ ] 分辨率和帧率满足题目。
- [ ] 开机前几帧已跳过或算法可容忍。
- [ ] 屏幕能显示任务、坐标、通信状态和错误。

## 视觉识别

- [ ] 单目标在正常光照下稳定识别。
- [ ] 目标丢失时状态明确。
- [ ] 误检有过滤策略。
- [ ] ROI 不会遮掉有效目标。
- [ ] 坐标方向和原点已验证。
- [ ] 物理单位换算已用标定点验证。

## 模型

- [ ] 模型文件存在且能加载。
- [ ] 置信度和 IOU 已现场测试。
- [ ] 误检和漏检策略明确。
- [ ] 模型失败时程序不会崩溃。

## 通信

- [ ] GND 共地。
- [ ] RX/TX 交叉。
- [ ] 波特率等参数一致。
- [ ] 发送端可被串口助手看到。
- [ ] 接收端能保存完整原始字节流。
- [ ] 解析器支持粘包和拆包。
- [ ] 协议、单位、频率、ACK/心跳已按题目确认。

## 下位机/闭环

- [ ] 单轴或单机构低速测试通过。
- [ ] 控制方向正确。
- [ ] 限幅、急停或保护存在。
- [ ] 目标 lost 时动作策略明确。
- [ ] 闭环不会因单帧异常大幅跳变。

## 现场准备

- [ ] 有最小相机测试脚本。
- [ ] 有最小视觉测试脚本。
- [ ] 有最小串口测试脚本。
- [ ] 关键参数集中可改。
- [ ] 保留稳定保守模式。
- [ ] 记录最后错误和最后通信内容。
