# 在线检索与来源分级

仅在 `SKILL.md` 的条件自动检索门触发时读取。本文件让检索缩短核验时间，而不是把公开案例直接变成最终工程。

## 触发与顺序

1. 先按题名、年份、视觉模式或关键词检索本地题库，提取相似模式和已知限制。
2. 再检索原始题面、官方 MaixPy/硬件资料或版本说明，核验会影响代码、接线、评分或安全的事实。
3. 仅在需要路线、调参范围、失败模式或验证经验时，再查公开项目、演示或教程。
4. 每次最多保留一条一级事实和两条必要的二级经验；只有外部资料会核验事实或改变候选路线时才输出研究卡，无关搜索结果不进入方案。

用户明确“禁止联网”时停止第 2、3 步。没有浏览能力时，输出“建议检索关键词 + 待确认事实”，继续基于用户材料和本地资料给出标明假设的候选路线。

## 来源分级

| 等级 | 可确认的内容 | 不可替代的内容 |
| --- | --- | --- |
| 一级：官方 MaixPy/硬件资料、原始题面、官方竞赛材料 | API、版本门槛、引脚/设备、规则、评分和限制 | 用户已确认的现场硬件、协议和安全策略 |
| 二级：公开 GitHub 项目、公开演示、高质量教程 | 路线选择、调参范围、常见失败、测试顺序 | MaixPy API 事实、题面要求、最终参数和最终协议 |

社区代码不得直接复制到正向工程；任何借鉴后的 MaixPy 代码仍须通过 `api-syntax-guard.md` 并回查官方文档。不得下载、转发或声称拥有未授权的数据集、模型、图片、密钥或私有链接。

## 检索关键词

- 题面：`<年份> <题号或题名> 全国大学生电子设计竞赛 题面`。
- API/硬件：`site:wiki.sipeed.com/maixpy <模块或 API> MaixPy`。
- 路线经验：`MaixCAM <视觉模式> <赛场约束>`，并把结果降级为二级经验。

检索完成后回到方案选择门禁；不能因为找到了案例而跳过评分、接口、性能或安全确认。
