# Skill Validation Scenarios

这些场景用于检查 skill 是否先收集配置、先给方案并等待选择、再给工程建议。它们不是固定项目模板。

## 通用空白输入

输入：“帮我用 MaixCAM Pro 做电赛视觉项目。”

期望：先询问题型/评分、对象与输出、相机、屏幕像素与方向、引脚/电平/下位机、供电/安全和场地；不立即给固定串口协议、UI 或代码。

## 2012 激光枪自动射击装置

题面模式：激光点、胸环靶、二维瞄准和报靶。

期望方案：颜色/高亮点 + 靶纸坐标系；候选方案比较曝光/ROI、物理标定、云台接口、失目标、限幅和激光安全。确认前不假定波特率、舵机引脚或报靶帧格式。

## 2019 巡线机器人

题面模式：路径巡查、黄色异物、条码/二维码图像保存。

期望方案：先将路径控制和巡查识别拆开；比较传统颜色/几何与码识别方案；询问相机姿态、航迹误差、存储、地面站 UI 和飞行安全。

## 2024 三子棋游戏装置

题面模式：九宫格、机械执行、人机回合、棋盘旋转和棋子移动复原。

期望方案：每回合棋盘重定位 + 观测/期望棋局差分；确认棋盘旋转范围、棋子材质、按钮事件、机械臂坐标、每步时限和 UI。不得只给固定九格像素坐标。

## 2025 单目视觉目标物测量

题面模式：固定单相机、黑框 A4 目标、距离与尺寸、误差和 5 秒时限。

期望方案：比较已知平面几何/标定与单应性路线；询问相机安装、单位、目标倾角、误差预算和禁止的非视觉测距。不得以未标定像素比例冒充物理测量。

## 2025 野生动物巡查系统

题面模式：格网、禁飞区、动物类别/数量/位置和地面站。

期望方案：规划与检测分层；模型前先检查数据和类别；无数据时进入 YOLO 采集/标注/训练计划。上报字段必须含格网、类别、数量、时间和置信度，协议与 UI 待确认。

## 2026 智能寻迹打靶协同系统

题面模式：循迹小车、移动靶、二维云台、激光追踪和画圆。

期望方案：先验证小车路径，再验证移动靶跟踪；明确相机/云台坐标标定、YOLO 双缓冲时延、低速限幅、急停和激光安全。不得把上一帧模型结果直接用于严格同步打靶。

## 陌生题

输入：一份不在题库中的新题面。

期望：先在 `nuedc-topic-coverage.md` 检索相似模式；没有直接适配时按目标、场地、输出、执行、评分和安全提出 2–3 条候选路线，标明假设并询问用户，而不是拒绝或强套旧题。
