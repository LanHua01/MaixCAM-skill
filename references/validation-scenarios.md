# Skill Validation Scenarios

这些场景用于检查 skill 是否先收集配置、先给方案并等待选择、再给工程建议。它们不是固定项目模板。

## 发布前执行规则

每个场景都以“输入 → 必须出现 → 禁止出现”判定。维护者记录实际输出片段或测试记录；任何禁止项出现即为失败，修复后重测。场景不在普通项目咨询时自动加载，仅在修改 Skill 后的回归自测使用。

## 通用空白输入

输入：“帮我用 MaixCAM Pro 做电赛视觉项目。”

期望：先询问题型/评分、对象与输出、相机、是否需要 UI 及屏幕像素/方向/UI 内容、引脚/电平/下位机、心跳/ACK/超时策略、供电/安全和场地；不立即给固定串口协议、UI 或代码。

- 必须出现：缺失配置问询。
- 禁止出现：最终协议、最终 UI 或完整工程代码。

## 核心后分支问询

输入：“固定场地上识别红色色块并显示中心坐标；无下位机、无执行机构、无独立 UI、无触摸、无模型。”

期望：先询问题目/评分、输出、相机安装与场地等核心配置，并将通信、执行、安全联锁、独立 UI/调参、模型标记为“无”。不再追问引脚、波特率、心跳、ACK、屏幕像素、触摸或数据集；随后进入紧凑工程选择。

- 必须出现：核心后分支、模块存在性和未启用能力的明确记录。
- 禁止出现：对已确认不存在模块继续追问细节，或将其写入最终工程。

## 快速 UART 最小验证

输入：“已确认 UART1 端口和引脚，请给一个只验证收发的最小 MaixPy 测试。”

- 必须出现：仅询问波特率、逻辑电平/共地和预期收发内容，给出独立最小测试与失败排查。
- 禁止出现：完整项目问询、最终协议、心跳/ACK 或执行机构代码。

## UI 与通信可靠性待定

输入：“题目需要视觉坐标上报，但尚未决定要不要屏幕显示，也没确定心跳或 ACK。”

期望：先询问是否需要 UI；需要时收集画面、状态、错误、性能和调参内容，不需要时给替代观测方案。通信方案至少比较无心跳、周期心跳、查询响应等候选，并把 ACK、序号、重发、超时和丢失策略作为用户确认项；不得自动发送心跳包。

- 必须出现：UI 是否需要、UI 内容或替代观测方式，以及可靠性机制比较。
- 禁止出现：未确认就发送默认心跳包、ACK 或重发逻辑。

## 板端调参 UI 与编码标记（可选）

输入：“我要识别场地上的 AprilTag，并希望在有触摸屏时现场调 ROI、阈值和显示内容；若本机没有触摸屏，也要能继续运行。”

期望：先确认标签家族/ID 或载荷、是否需要角点/物理坐标、打印尺寸、距离/光照、屏幕分辨率和方向、触摸/按键/替代输入是否实际可用。给出至少“无 UI/串口观测”和“可选运行/调参 UI”两种方案；说明 UI 仅调白名单视觉参数，不能默认修改引脚、协议、限幅或安全策略。比较 QR 与 AprilTag 时说明输出差异，不将其混为一类。

- 必须出现：可选 UI 的启用条件、无输入时的降级方式、编码标记的坐标映射/标定问题，以及相机/视觉/控制发布三类速率记录。
- 禁止出现：固定 320×240 布局、伪造触摸/按键 API、把 QR 与 AprilTag 无条件互换，或用相机 FPS 声称闭环性能达标。

## 已有工程的 UART 联调

输入：“现有 MaixPy 项目已经用 UART1 与下位机通信，波特率和协议不可修改；现在目标坐标会跳变，请帮我修复。”

期望：先阅读现有视觉、串口与下位机解析代码，保留已确认接口；询问跳变条件和验收动作。先建议固定测试包、候选/确认/缓存/丢失状态与独立串口验证；不重写协议、不擅自改引脚或直接引入第二套主循环。

- 必须出现：先读取已有工程、保留既有接口和独立验证顺序。
- 禁止出现：未阅读工程即重写协议或主循环。

## MaixVision 项目运行异常

输入：“MaixVision 里单文件能显示相机，但完整项目导入模块和模型后运行失败，编辑器还有红线。”

期望：先收集 MaixVision/固件版本、完整日志、运行方式和资源部署情况；区分电脑端补全、完整项目资源和板端运行时。先验证项目入口与设备侧资源，再用最小相机显示恢复模块；不得把 IDE 红线直接断言为板端错误，或把电脑绝对路径写进部署方案。

- 必须出现：版本/日志问询、完整项目运行与设备侧资源检查。
- 禁止出现：把编辑器红线直接当作板端错误，或使用电脑绝对路径。

## MaixVision 一致性与可复现调试

输入：“编辑器里最新代码能看到，但设备仍像在运行旧逻辑；打包后模型又找不到，且我只在办公室近距离正常光照下测过。”

期望：先建立源码快照、工程/资源清单、运行方式、部署/包标识、板端运行身份和分阶段日志的证据链；明确区分运行当前文件、运行完整项目、打包/安装。要求模块接口契约覆盖参数、单位/范围、返回字段、错误与配置键，并先独立验证再恢复全链路。要求场景回归至少覆盖远近、光照/反光、遮挡/空场、背景/目标切换和资源缺失；日志不含电脑绝对路径或凭据。

- 必须出现：调试证据记录、接口契约、部署清单、阶段/模块/运行身份/配置或输入摘要，以及场景回归矩阵。
- 禁止出现：仅凭编辑器或 Git 状态断言设备版本、仅在单一场景调阈值就声称泛化、无资源清单就重打包，或把单次日志当作完整部署证明。

## 完整工程目录交付

输入：“我选择传统颜色识别方案。请生成一个项目，我在 MaixVision 打开文件夹后就能直接开始调试。”

期望：先确认核心配置与模块存在性；再仅对存在的模块补充相机、屏幕、引脚或通信配置，按风险选择紧凑工程或完整闭环工程，输出带唯一 `main.py` 和必要文件的完整目录树及所有文件。单一颜色识别且无下位机、UI、模型或独立风险时，使用仅含 `main.py` 的紧凑工程，不创建空通信/显示/模型模块或无效测试。使用相对路径，首次启动显示画面和状态；提示用户运行完整项目而非当前文件。若模型或资源未提供，交付可运行的安全降级/验证工程并明确缺失项。

- 必须出现：工程规模、唯一入口、相对路径、完整项目运行、可观察首启状态、每个文件的生成/不生成理由，以及题目必需/可选未启用/明确排除能力。
- 禁止出现：只给零散片段、电脑绝对路径、虚构模型资源，或为简单任务生成空通信/UI/模型模块。

## 在线检索与来源分级

输入：“这是一份陌生的视觉题，我只给出题名和图片；请帮我确定可行方案，可以联网。”

期望：先按题名/年份/视觉模式检索本地题库；题面规则、MaixPy API、硬件或版本需要核验时再在线检索。方案矩阵前给出研究卡，明确来源链接、一级官方事实、二级公开经验、对候选路线的影响和待确认项。公开案例不可直接成为最终 API、接线、协议或评分结论。

- 必须出现：本地检索、分级来源、研究卡和待确认项。
- 禁止出现：把社区代码当 MaixPy 官方依据，伪造链接/数据集/模型，或以搜索结果覆盖用户已确认约束。

## 禁止联网或无浏览能力

输入：“题目陌生，但本次禁止联网；请先给方案。”

期望：检索本地题库，给出建议检索关键词、待确认事实与标明假设的候选路线；继续完成本地可得的拆题，不声称已经查到外部资料。

- 必须出现：本地分析、检索关键词和待确认项。
- 禁止出现：虚构在线来源，或因不能联网而拒绝分析。

## 紧凑工程避免过度实现

输入：“我只需 MaixCAM Pro 识别固定场地的红色色块并在屏幕显示中心坐标；无下位机、无触摸、无模型、无闭环。”

期望：选择仅含 `main.py` 的紧凑工程；使用 `find_blobs()`、ROI 和曝光验证等传统视觉路线。仅在参数独立调节、拆分降低复杂度或独立风险存在时才增加 `config.py`、任务文件或测试。说明没有通信、独立 UI、模型和闭环，因此不生成对应模块。

- 必须出现：紧凑工程、传统视觉优先、文件必要性理由。
- 禁止出现：UART 协议、心跳/ACK、YOLO、空 `communication.py`、空显示模块或无关测试。

## 检索只在影响决策时出现

输入：“题面已明确为固定红色色块中心坐标显示，MaixPy 版本和相机配置已确认；不需要外部案例。”

期望：不输出研究卡或外部来源；直接基于已确认题面选择传统视觉紧凑工程。若陌生题确需核验 API 或题面规则，研究卡最多保留一条一级事实和两条必要二级经验。

- 必须出现：仅在影响决策时检索的判断。
- 禁止出现：为已确认简单任务强行搜索或堆叠无关来源。

## 复杂闭环的最简选择

输入：“移动靶云台需要类别识别和低速安全闭环；传统颜色可能受背景干扰，MaixPy 版本与模型资源待确认。”

期望：先核验题面、目标板版本和模型资源，比较传统颜色/几何、YOLO 或融合路线；先淘汰不满足语义识别、安全、时延和性能余量的方案，再在可行方案中比较依赖、标定、模块数量和调试成本。只有确认模型确有必要时才进入 YOLO 数据/部署计划和完整闭环工程。

- 必须出现：先合格再最简、性能/安全/复杂度比较和模型必要性判断。
- 禁止出现：因存在公开案例就直接选 YOLO，或在未确认模型/接口前生成最终闭环代码。

## YOLO 版本与数据不足

输入：“请用 YOLO26 做动物识别；我的 MaixPy 版本未知，也没有数据集。”

- 必须出现：先询问/核验 MaixPy 版本，说明 YOLO26 的版本门槛，并进入数据采集、标注、训练和部署计划。
- 禁止出现：未核验版本就生成 `nn.YOLO26` 最终代码，或声称已有可用数据集/模型。

## 中文代码注释

输入：“请生成完整 MaixCAM Pro 工程，所有注释使用中文。”

期望：Python 关键字、MaixPy API、文件名、变量名和协议字段保持原有英文；对模块职责、接线/硬件假设、坐标单位、安全状态、时序和调参写必要中文注释，不添加逐行翻译式冗余注释。

- 必须出现：中文的必要工程注释。
- 禁止出现：中文化 API/标识符，或逐行翻译式冗余注释。

## 工程级关键注释覆盖

输入：“已确认 MaixCAM Pro、相机和 UART1；请生成颜色目标上报工程。任务需要 `candidate`、`confirmed`、`cached`、`lost` 状态，坐标单位为像素，发布频率已确认；所有代码采用工程级关键注释。”

期望：在不改变已确认接口的前提下，生成的完整工程对模块职责和硬件假设、关键阈值/频率的单位与影响、任务输入输出和坐标/失效语义、状态转换目的、发布限频或超时安全理由写必要中文注释。若选用模型双缓冲，还要说明其结果时序影响。最小测试仅说明验证条件、预期现象和关键失败条件。

- 必须出现：上述相关位置的中文关键注释，且 API、标识符和协议字段保持英文。
- 禁止出现：逐行解释 `import`、赋值或循环；未确认时虚构协议/引脚；为凑注释而写与工程无关的硬件或模型说明。

## 目标板性能不达标

输入：“MaixCAM Pro 上完整项目只有较低帧率，打开 YOLO 和屏幕叠加后还会偶发卡死。”

期望：先索取板卡/固件、相机与模型配置、目标 FPS/时延、性能报告、运行时长和日志；按相机、视觉、模型、UI/串口分层建立基线。输出问题、证据、可能瓶颈、候选优化、精度/时序代价和回归测试，不把降低分辨率或开启双缓冲当作无条件答案。

- 必须出现：目标板实测配置、分层基线和优化代价/回归测试。
- 禁止出现：无实测证据就断言性能达标，或无条件降低分辨率/开启双缓冲。

## 2012 激光枪自动射击装置

题面模式：激光点、胸环靶、二维瞄准和报靶。

期望方案：颜色/高亮点 + 靶纸坐标系；候选方案比较曝光/ROI、物理标定、云台接口、失目标、限幅和激光安全。确认前不假定波特率、舵机引脚或报靶帧格式。

- 必须出现：候选方案、曝光/标定、失目标和激光安全。
- 禁止出现：确认前固定波特率、舵机引脚或报靶帧格式。

## 2019 巡线机器人

题面模式：路径巡查、黄色异物、条码/二维码图像保存。

期望方案：先将路径控制和巡查识别拆开；比较传统颜色/几何与码识别方案；询问相机姿态、航迹误差、存储、地面站 UI 和飞行安全。

- 必须出现：路径控制与巡查识别分层，以及至少两条识别路线比较。
- 禁止出现：未确认相机姿态和安全约束就给最终飞行/控制实现。

## 2024 三子棋游戏装置

题面模式：九宫格、机械执行、人机回合、棋盘旋转和棋子移动复原。

期望方案：每回合棋盘重定位 + 观测/期望棋局差分；确认棋盘旋转范围、棋子材质、按钮事件、机械臂坐标、每步时限和 UI。不得只给固定九格像素坐标。

- 必须出现：每回合重定位、棋局差分和机械接口/时限问询。
- 禁止出现：固定九格像素坐标作为最终方案。

## 2025 单目视觉目标物测量

题面模式：固定单相机、黑框 A4 目标、距离与尺寸、误差和 5 秒时限。

期望方案：比较已知平面几何/标定与单应性路线；询问相机安装、单位、目标倾角、误差预算和禁止的非视觉测距。不得以未标定像素比例冒充物理测量。

- 必须出现：至少两条测量路线、标定与误差预算。
- 禁止出现：用未标定像素比例声称物理测量结果。

## 2025 野生动物巡查系统

题面模式：格网、禁飞区、动物类别/数量/位置和地面站。

期望方案：规划与检测分层；模型前先检查数据和类别；无数据时进入 YOLO 采集/标注/训练计划。上报字段必须含格网、类别、数量、时间和置信度，协议与 UI 待确认。

- 必须出现：数据/类别问询或 YOLO 采集标注计划，以及上报字段候选。
- 禁止出现：未给数据就声称模型可部署，或固定协议/UI。

## 2026 智能寻迹打靶协同系统

题面模式：循迹小车、移动靶、二维云台、激光追踪和画圆。

期望方案：先验证小车路径，再验证移动靶跟踪；明确相机/云台坐标标定、YOLO 双缓冲时延、低速限幅、急停和激光安全。不得把上一帧模型结果直接用于严格同步打靶。

- 必须出现：分步验证、坐标标定、双缓冲时序和安全措施。
- 禁止出现：将上一帧模型结果直接作为严格同步打靶指令。

## 陌生题

输入：一份不在题库中的新题面。

期望：先在 `nuedc-topic-coverage.md` 检索相似模式；没有直接适配时按目标、场地、输出、执行、评分和安全提出 2–3 条候选路线，标明假设并询问用户，而不是拒绝或强套旧题。

- 必须出现：相似模式检索或 2–3 条候选路线、明确假设和关键问询。
- 禁止出现：拒绝处理，或把历史题协议/坐标直接套入新题。
