# MaixPy API and Syntax Guard

在输出或修改 MaixPy 代码前执行本清单；具体 API 与硬件事实仍以 `maixcam-pro-api-notes.md` 链接的官方资料为准。

## 运行时边界

- 只导入已由官方资料确认的 `maix` 模块；不混入 OpenMV、K210、树莓派、桌面 Python 或 PC OpenCV 专属接口。
- 区分 MaixVision 的开发环境动作和 MaixPy 的板端运行时代码。
- 固件、板卡或方法签名存在差异时，写明假设；不确定时先输出最小验证代码，不编造 API。

## 输出前检查

- 检查 `import`、类名、方法名、参数和返回值是否符合官方资料。
- 检查工程代码采用中文的工程级关键注释；保留 Python、MaixPy API、文件名、标识符和协议字段的英文拼写。注释服务于接口、决策和风险，不逐行复述赋值、循环或显而易见的语法。
  - 文件或模块入口：说明职责、依赖的板卡/设备或资源假设，以及不能满足时的降级入口。
  - `config`：为会影响结果的配置说明单位、适用范围或来源（已确认时）、识别/性能/安全影响；不要把未经确认的数值伪装成固定推荐值。
  - 跨模块公共函数、类或任务接口：说明输入、输出、坐标/单位、有效性和错误/丢失语义；内部显而易见的小助手无需逐个写注释。
  - 状态机和缓存：说明 `candidate`、`confirmed`、`cached`、`lost` 等转换条件及其目的，避免单帧结果直接驱动执行机构。
  - 非直观时序与安全：说明 `dual_buff` 延迟、发布限频、超时、限幅、急停或安全降级“为什么这样做”，并标出会影响闭环时序或性能的取舍。
- 检查 `if`、`for`、`while`、`def`、`class` 的冒号和缩进，括号/字符串闭合，以及变量在使用前已定义。
- 检查图像格式、相机分辨率、模型输入、坐标方向和单位在调用链中一致。
- 检查跨模块函数/类的参数、类型、单位/范围、返回字段、错误状态和配置键已有明确契约；接口变化时先更新实现、调用方和独立契约测试，再运行完整项目。
- 检查 UART 文本/字节发送方式、打包字段数、字节序和校验与已确认协议一致。
- 检查轮询接收与回调接收没有混用；闭环模型推理已处理 `dual_buff` 的一帧延迟。
- 检查异常、模型失败、低置信度和目标丢失不会直接触发危险动作。
