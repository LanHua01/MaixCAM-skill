# Vision Archetypes

本文件是陌生题和历史题检索的**首要路由入口**。先按当前题面的目标、评分、场地、输出和安全约束匹配视觉原型，再查看 `verified-vision-topics.md` 中的已核验历史题，最后回到当前题面确认。`topic-title-archive.md` 只能生成公开检索键，不得按题名决定方案。

## 路由总表

| 当前任务证据 | 首选原型 | 升级条件 |
| --- | --- | --- |
| 颜色、亮点、灯光状态 | Blob / LAB | 光照与材质导致阈值无法稳定分离 |
| 黑线、边缘、轨迹、停止线 | Line / Edge | 交叉/遮挡需要状态或目标语义 |
| 矩形、圆、角点、已知尺寸 | Geometry / 标定 | 不规则像素区域或类别语义不可替代 |
| 编号、固定地标、姿态 | QR / AprilTag | 题面禁止标记或可见性不足 |
| 类别与位置 | Detection | 还需要方向、像素掩膜或跨帧身份 |
| 旋转物体方向 | OBB | 普通框加几何主轴已经足够时不升级 |
| 不规则区域/道路轮廓 | Segmentation | Detection + 轮廓后处理已经满足评分时不升级 |
| 跨帧身份、去重、连续控制 | Tracking | 简单门限和短时缓存足够时不升级 |

能力、版本和降级路线见 `maixpy-vision-capabilities.md`。

## 二维码、AprilTag 与编码标记

适用：巡查站点确认、任务编号读取、固定地标定位、靶标 ID 关联、视觉坐标系初始化。

- 先确认输出是 QR 载荷、AprilTag ID、四角点、旋转角、测距还是位姿；不同输出决定是否需要相机标定和坐标映射。
- QR 更适合承载文本/编号等内容；AprilTag 更适合已知 ID 的定位关联。不能因两者都属于方形标记而替换题目要求。
- 先在完整分辨率画面上确认可见性，再比较 ROI、缩放识别和原图坐标回映；若在缩放图检测，必须明确恢复角点/中心到原图或物理坐标系的比例与误差。
- 读取内容或 ID 后仍需候选、确认、缓存和丢失状态；单帧错误标记不得直接触发危险动作或改写任务状态。
- 若涉及距离或姿态，先确认标签实际尺寸、相机内参、安装方式、允许标定和误差指标；没有标定不得把像素尺寸或角点当作物理测量结果。

必问：标记类型/家族、载荷或 ID 表、打印尺寸与材质、最远距离、倾角/运动范围、是否需要角点或物理坐标、遮挡/误读处理、输出和安全动作。

## 激光点追踪、打靶与自动瞄准

适用：识别激光点、目标靶、移动靶、黑框/A4 靶纸，控制云台或执行机构。

- 先分离“靶纸/边框定位”和“高亮光点定位”；两者可能需要不同曝光。
- 输出必须带时间戳、可见性、面积/置信度和坐标系；闭环先低速、限幅、单轴验证。
- 移动靶或双缓冲模型场景要明确视觉时延、控制周期和图像/结果关联，不能把上一帧结果当当前帧。
- 激光必须有急停、失目标策略和人眼安全约束；单帧跳变不直接驱动云台。

必问：光点/背景颜色、相机是否固定、目标坐标系、控制频率、补光/固定曝光、执行机构限幅和急停。

## 黑框、靶纸、几何与单目测量

适用：A4 黑框、圆/矩形/三角形尺寸、目标距离、平面姿态、物理坐标输出。

- 先建立相机、靶纸和场地坐标系；透视明显时使用标定、单应性或明确的几何模型。
- 误差预算必须覆盖像素量化、焦距/安装误差、距离变化、目标倾斜和边缘提取误差。
- 固定相机、已知边框和已知尺寸可以降低问题复杂度；不得在题目禁止时改用非视觉测距。
- 按题目时限将“检测、标定映射、显示”拆分测试，避免只在离线图上验证。

必问：相机固定方式、目标尺寸/距离范围、允许标定次数、倾角范围、物理单位、误差/时间指标、外部传感器限制。

## 线、边缘与轨迹

适用：小车/无人机巡线、黑框边缘、路径方向、交叉线和停止线。

- 控制误差优先来自有限 ROI 内的回归线、边缘或连通区域，不在每帧堆叠多套全图检测。
- 把普通线、交叉、停止线、坡道和丢线定义成不同状态；跨状态前要求连续确认。
- 先实现低速基础循迹，再增加预瞄 ROI、交叉决策或视觉目标；新增能力必须可关闭。

必问：线宽/颜色、相机姿态、速度、交叉规则、停止线、丢线动作、控制器输入和时限。

## 棋盘、格点与三子棋

适用：九宫格、棋盘、机械臂落子、人机对弈和棋子被移动后的复原。

- 每次回合前重定位棋盘，不能假设棋盘始终固定；旋转或位移题先恢复棋盘坐标系再映射格点。
- 维护“期望棋局”和“观测棋局”的状态差分，识别被移动棋子后输出复原动作，而不是只识别当前格子。
- 机械执行前校准格子中心、取子区和安全路径；将放置成功、棋子颜色、越线风险分开验收。
- 人机回合应由题目指定按钮/事件触发，且将检测、决策、执行纳入单步时限。

必问：棋盘旋转范围、棋子颜色/材质、落子机构、格子编号或物理坐标、回合触发方式、每步时限和复原要求。

## 巡线、巡查、格网与路径视觉

适用：小车循线、无人机巡查、格网覆盖、禁飞区、交叉线、停止线和巡查目标事件。

- 近处 ROI 用于控制，远处 ROI 用于预判；交叉、断线、坡道、停止线和丢线是独立状态，不是普通线误差。
- 格网巡查须将地图坐标、禁行单元、覆盖状态、目标事件和上报记录分开保存。
- 无人机/车辆闭环先验证路径，再叠加识别；通信丢失、视觉丢失和执行器故障都必须有保守策略。
- 目标识别结果应绑定格网/路径位置和时间，避免跨格重复统计。

必问：线颜色/宽度、相机姿态、交叉/停止线、格网尺寸、禁区规则、控制误差形式、覆盖判定与安全限制。

## 动物、火焰、车辆和复杂目标检测

适用：需要类别语义、复杂外观、多姿态或多目标计数的巡查/报警任务。

- 优先判定传统颜色/几何是否足够；仅在复杂类别识别确有需要时使用 YOLO。
- 无数据时先定义类别、现场采集范围、标注和训练/验证划分；不得声称存在未提供的数据集或模型。
- 模型输出经 ROI、尺寸、类别、格网位置与时序稳定性过滤后才上报或动作。
- 计数要定义同一目标的去重依据、停留时间和跨格移动策略。

必问：类别表、已有图片/标注/模型、模型许可、误检与漏检代价、目标尺寸/姿态、离线/实时要求和上报字段。

## OBB、分割与跨帧追踪

适用：旋转目标方向、不规则像素区域、道路/禁区掩膜、跨帧目标身份和计数去重。

- 只有方向本身影响得分时才比较 OBB；普通检测框加几何主轴已足够时不引入旋转模型。
- 只有像素级边界不可替代时才比较分割；规则道路、靶框或颜色区域优先传统视觉。
- Tracking 建立在可用检测结果之上，先验证单帧检测，再验证遮挡、ID 切换、重复计数和结果时延。
- 模型路线必须先核对版本、数据和目标板性能；任何未经板端实测的性能结论标为“待实测”。

必问：方向/掩膜/身份是否属于评分项、目标数量与遮挡、允许时延、最低 MaixPy 版本、模型与数据状态、降级动作。

## 多节点与通信边界

适用：多个视觉节点、地面站、下位机、云台/机械臂/小车联动。

- 每个节点输出应说明来源、时间戳、坐标系、有效性和置信度。
- 协议由题目和用户确认；不能因为历史题使用过某种帧格式就默认复用。
- 串口、网络和无线链路各自独立测试；下位机不在线时视觉模块仍可显示与记录。

## 陌生题处理

1. 重新阅读当前题面，提取评分、时限、器件限制、动作和失效边界。
2. 使用本文件匹配视觉原型，再查询 `verified-vision-topics.md`；只有需要公开检索键时才查 `topic-title-archive.md`。
3. 若无直接适配，依据“目标—场地—输出—执行—评分—安全”提出候选路线；没有真实取舍时只给最简合格路线。
4. 明确题面事实、官方能力事实、工程经验和待确认假设；最终仍以当前题面和板端实测为准。
