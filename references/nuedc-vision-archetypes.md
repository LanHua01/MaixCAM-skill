# NUEDC Vision Archetypes

本文件归纳电赛和相近光电视觉题中反复出现的视觉工程模式。使用时不要机械套用；先问题目评分指标、场地、接口和硬件限制。

## 激光点追踪与自动瞄准

常见目标：

- 识别红/绿激光点。
- 让一个光斑跟踪另一个光斑。
- 控制云台、绳驱、步进电机或舵机把光斑移动到目标。
- 沿黑框、A4 靶纸边线或指定路径运动。

工程重点：

- 曝光和增益比算法更关键。
- ROI 能显著减少杂光误检。
- 光斑坐标必须带可见性、置信度或面积信息。
- 单帧跳变要滤波或二次确认。
- 视觉输出应是误差或坐标，闭环控制通常交给下位机或控制模块。

使用前必须问：

- 光斑颜色和背景颜色是什么？
- 相机固定还是随执行机构运动？
- 输出像素坐标、物理坐标还是控制误差？
- 控制频率和下位机接口是什么？
- 是否允许补光、遮光、固定曝光？

## A4 靶纸、黑框、边线、矩形

常见目标：

- 找黑色矩形框四角。
- 沿矩形边线运动。
- 识别靶纸相对相机的位置和角度。
- 将激光点约束到边线或框内指定位置。

工程重点：

- 黑框识别和激光识别可能需要不同曝光/阈值，必要时分阶段处理。
- 先建立靶纸坐标系，再输出目标坐标。
- 透视明显时需要标定或单应性映射。
- 赛场灯光变化会影响黑色阈值，尽量用几何约束过滤。

使用前必须问：

- 靶纸尺寸和黑框宽度是否固定？
- 相机视角是否垂直靶纸？
- 是否需要四角坐标，还是只需要中心/边线误差？
- 是否允许先标定一次再运行？

## 棋盘、格点、三子棋

常见目标：

- 识别九宫格或棋盘边界。
- 判断棋子类型和格子占用状态。
- 输出机械臂/机构落子坐标。
- 做人机对弈状态判断。

工程重点：

- 先定位棋盘坐标系，再映射每个格点。
- 格子状态比单个物体检测更重要。
- 棋子识别可用颜色、形状、模板或模型。
- 机械执行前必须做格点坐标校准。

使用前必须问：

- 棋盘是否固定？
- 棋子颜色、形状和材质是什么？
- 输出给机械臂的是格子编号还是实际坐标？
- 是否需要 AI 对弈逻辑，还是只做视觉识别？

## 摄像测量与多节点坐标估计

常见目标：

- 多摄像节点同时观察运动点。
- 从多角度估计位置、速度或轨迹。
- 通过网络或串口汇聚到终端节点。

工程重点：

- 时间同步和坐标系统一比单帧识别更重要。
- 每个节点输出应包含时间戳、节点编号、目标坐标和有效性。
- 需要明确相机安装位置和几何关系。
- 如果通信延迟大，应避免下位机把旧坐标当新坐标。

使用前必须问：

- 有几个摄像节点？
- 节点之间如何通信？
- 是否需要时间同步？
- 坐标系如何定义？

## 小车巡线、巡查与路径视觉

常见目标：

- 找线、路口、边界、障碍。
- 识别巡查目标或场地标志。
- 输出方向误差、路径状态或目标事件。

工程重点：

- ROI 分层：近处用于控制，远处用于预判。
- 输出控制误差比输出所有像素点更实用。
- 巡线算法要有丢线处理、路口处理、反光处理。
- 速度越快，越需要限制视觉处理复杂度。

使用前必须问：

- 线的颜色、宽度和背景是什么？
- 相机安装角度和高度是多少？
- 输出给小车的是角度误差、横向误差还是路径状态？
- 是否存在交叉口、断线、障碍物？

## 火焰、动物、车辆、物体检测

常见目标：

- 使用 YOLO 或分类模型识别目标。
- 输出目标类别、框、中心点、置信度。
- 与巡查、瞄准、报警或灭火机构联动。

工程重点：

- 目标检测模型需要现场误检策略。
- 置信度阈值不能只看离线测试，要看赛场背景。
- 模型输出最好经过 ROI、尺寸、时序稳定性过滤。
- 若目标颜色明显，传统视觉可作为模型前置或后验约束。

使用前必须问：

- 模型是否已有，格式是否为 MaixCAM 可用的 `.mud`？
- 需要识别类别还是只需要目标中心？
- 误检和漏检哪个代价更高？
- 是否有实时性要求？

## 通用电赛视觉判断

优先级建议：

1. 能用固定几何解决，不先上 AI。
2. 能用颜色和 ROI 稳定解决，不先上复杂模型。
3. 需要类别语义或复杂外观时再使用 YOLO。
4. 输出应贴近控制需求，而不是输出算法内部细节。
5. 每个目标都要有 `visible/lost/cached` 或等价状态。
