# 可选板端运行/调参 UI 配置档

仅在用户明确需要现场观测或调参，并已确认屏幕尺寸/方向以及触摸、按键或替代输入至少一种可用时读取本文件。它是现有工程的可选增强，不替代方案选择、协议确认、独立测试和安全状态机。

## 选择与边界

| 配置档 | 适用条件 | 内容 | 不做什么 |
| --- | --- | --- | --- |
| 无 UI / 外部观测 | 无屏、控制时限紧，或用户不需要板端显示 | 串口日志、保存结果或上位机观测（由用户确认） | 不伪造显示或输入模块。 |
| 运行监视 | 有屏幕，需要现场确认状态 | 画面叠加、识别/通信/错误状态和三类性能指标 | 不默认允许修改参数。 |
| 运行 + 调参 | 有屏且确认至少一种输入，用户需要现场调参 | 运行页与调参页、参数范围、恢复默认和变更记录 | 不允许 UI 改变引脚、协议、安全限幅、急停或模型文件。 |

不得固定屏幕像素、页面布局或控件数量。显示层从配置读取实际宽高和方向；未确认输入 API 时，先生成独立输入探针或采用用户确认的替代方式。

## 最小页面与参数规则

- **运行页**：只显示用户要求的相机/叠加画面、目标状态、错误、通信状态，以及相机采集、有效检测、控制发布三类实测速率。显示“待实测”而非伪造性能结论。
- **调参页**：仅暴露题目相关、可回退的白名单视觉参数，例如 ROI、阈值、置信度、面积过滤、显示开关和 UI 刷新上限。每项给出单位、范围、步长、当前值和恢复值。
- **状态隔离**：输入事件写入候选配置；在主循环边界完成范围检查后原子应用。参数变更必须保留可观察记录，失效时恢复上一次已验证配置。
- **安全隔离**：UI/输入故障只影响显示或调参；执行机构继续走既有候选、确认、限幅、超时和急停路径。禁止由单帧画面、点击事件或参数滑动直接驱动危险动作。

## 调度与性能

- 视觉和安全控制优先；UI 使用限速刷新，不要求每帧重绘。将 UI 绘制、日志和串口调试打印视为可测负载。
- 分别测量采集、有效检测和控制发布频率；控制发布频率以实际经过状态机释放的结果为准，不以 `cam.read()` 次数或串口写入次数替代。
- 若页面叠加导致帧率、时延或稳定性超标，依次减少绘制/刷新、缩小调试信息、关闭 UI，再验证视觉精度和闭环时序。优化必须保留前一份可运行配置。

## 独立验证顺序

1. `display_test`：核对实际分辨率、方向、显示内容和无 UI 降级。
2. `input_test`：仅在硬件已确认时核对触摸/按键/替代输入的事件、坐标映射和失效行为；不要把 IDE 的模拟现象当作板端事实。
3. `tuning_test`：确认参数白名单、上下限、单次变更、恢复和记录；不接执行机构。
4. `performance_test`：比较 UI 关闭、运行监视、运行+调参三种状态的速率、时延和连续运行稳定性。
5. 低速闭环联调：确认 UI 打开、输入丢失或参数恢复时，急停、限幅和失效策略仍然有效。

## MaixPy 语法要求

触摸、按键和显示 API 必须以当前目标板及 MaixPy 版本的官方文档为准。生成正向代码前读取 `api-syntax-guard.md`；无法核验的输入接口只生成最小探针和待确认项，不从 OpenMV、K210 或桌面框架迁移调用方式。
